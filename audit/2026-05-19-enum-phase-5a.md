Standard `dd_accounting_system` shape. RLS enabled. Public SELECT policy applied.

**Column change:**
- `eos_todos.status` altered from `public.eos_todo_status` to `text NULL`.
- USING `status::text` — safe, all values byte-identical.
- Default changed to `'Open'::text`.
- FK `eos_todos_status_fkey` added → `dd_eos_todo_status(value)` ON UPDATE CASCADE ON DELETE RESTRICT.

**View DROP + RECREATE — `v_client_eos_summary`:**
The view had a pg_depend on `eos_todo_status` and was dropped before the column type change, then recreated with an identical definition. Discovery during session: the view contains a dead condition `WHERE status = 'Done'` — `'Done'` is not an `eos_todo_status` enum value and has never matched any row. Preserved byte-identical (the condition is harmless). Tracked as BUG-040 for a future fix.

**Legacy enum retained:**
`public.eos_todo_status` retained in `public` schema with `COMMENT ON TYPE` retention notice (Phase 5Z cleanup). Not archived in this migration.
