## What shipped

A new Supabase Edge Function (`generate-certificate-pdf`) that generates a
branded PDF certificate when a learner clicks Download on the Academy
Certificates page (`/academy/certificates`).

**No DB migrations.** No schema changes. No RLS changes.
`storage_path` and `public_url` columns already existed on `academy_certificates`.

---

## Changes

### Edge function — `supabase/functions/generate-certificate-pdf/index.ts`
- New function, follows canonical `invite-user` auth pattern
- Validates caller JWT and confirms caller owns the certificate (or is Vivacity staff)
- Fast path: if `storage_path` already set, re-signs and returns URL without regenerating
- PDF generation: fetches Angela's PNG template from `doc-templates/academy/certificate-template-a4.png`, uses `pdf-lib` to overlay five text fields (certify phrase, recipient name, completion phrase, course title, issue date + cert number) onto an A4 landscape canvas
- Uploads PDF to `academy-certificates/{tenant_id}/{certificate_number}.pdf`
- Writes `storage_path` and `public_url` (5-year signed URL) back to `academy_certificates`

### Frontend — `src/pages/academy/AcademyCertificatesPage.tsx`
- Download button now calls the edge function when `public_url` is null
- Shows spinner + "Generating..." while in flight
- Updates local React Query cache on success so button stays active without page refresh
- Toast on error

### Storage asset (manual upload)
- `doc-templates/academy/certificate-template-a4.png` — Angela's branded A4 landscape certificate template (2000×1414px PNG, private bucket)

## Certificate number format

`VA-{YYYY}-{000001}` — auto-generated by `generate_certificate_number()` DB function using `academy_certificate_seq` sequence. Numbers are permanent — deletions do not recover sequence values.
