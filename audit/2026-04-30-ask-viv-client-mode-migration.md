## Findings

- **Table created cleanly.** `ai_client_query_usage` shipped as a single transactional migration. Verified via Supabase MCP: table exists, RLS enabled, both policies present, trigger present.

- **Pre-migration audit surfaced two spec corrections.** Codebase review before Lovable prompts ran identified that the original spec referenced `tenant_members` in the RLS policy — `tenant_users` is the canonical membership table (785 rows vs 391 in the legacy `tenant_members`). Corrected to a simpler `user_id = auth.uid()` self-read policy; no tenant join needed for a per-user counter. Also corrected staff function from `is_vivacity()` to `is_vivacity_internal_safe(auth.uid())` to match current `ai_interaction_logs` policy standard.

- **Five design decisions locked before migration.** Cap scope (per-user-global, not per-tenant), `ON DELETE CASCADE` on both FKs, self-read RLS (no membership join), `is_vivacity_internal_safe(auth.uid())` for staff policy, UTC day boundary (`CURRENT_DATE`) for reset semantics. All five confirmed before Prompt 2 (implementation plan).

- **No off-peak window required.** FK targets (`users`: 487 rows, `tenants`: 405 rows) trivial — sub-second migration. `VALIDATE CONSTRAINT` split not needed.

- **UNIQUE constraint covers index.** Postgres creates an index for `UNIQUE (user_id, query_date)` implicitly; the original spec included a redundant explicit `CREATE INDEX` which was correctly omitted.

- **Trigger verified.** `update_ai_client_query_usage_updated_at` confirmed present via `pg_trigger`. The `RI_ConstraintTrigger_*` entries also visible are Postgres-internal FK enforcement triggers — expected.

## Decisions

- **Cap scope: per-user-global.** `UNIQUE (user_id, query_date)`. In v1, one tenant per user, so global = per-tenant. If multi-tenant support is added (open question 1 in spec), unique key becomes `(user_id, tenant_id, query_date)`.
- **UTC reset semantics confirmed.** `CURRENT_DATE` evaluates in DB session timezone (UTC). Effective reset ~10:00 AEST. Documented in spec as known UX quirk; copy reads "Resets daily."
